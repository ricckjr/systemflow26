# Web Quality — SystemFlow

Objetivo
- Revisar qualidade de frontend: layout, largura de containers, grid, spacing, tipografia, responsividade, consistência visual e legibilidade.

Quando usar
- Revisar dashboards
- Revisar páginas React
- Revisar componentes de UI

Instruções
- Detectar espaço vazio excessivo.
- Detectar containers muito estreitos ou mal dimensionados.
- Detectar inconsistências visuais e wrappers desnecessários.
- Nunca alterar lógica de negócio ou contratos de dados.
- Respeitar tokens do design system, Tailwind e lucide-react.
- Seguir .trae/project_rules.md, .claude/napkin.md e .interface-design/system.md.

Formato de resposta
- Resumo
- Problemas Visuais
- Problemas Estruturais
- Melhorias Recomendadas
- Implementação Segura

Notas
- Responder em Português Brasileiro.
- Não duplicar estado; se sugerir mudanças, focar em estilo/estrutura.
